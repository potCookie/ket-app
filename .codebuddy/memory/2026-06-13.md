## Phase 5 Bugfix (12:36-12:48) — Pinia 响应式绑定 + 数据库种子数据

### 根因: `const x = store.ref` 捕获初始值(null)快照，非响应式

在 `<script setup>` 中，Pinia 代理会 unwrap ref，所以 `const todayData = taskStore.todayData` 捕获的是**当前值**(null)的快照。即使后续 `fetchTodayTask()` 更新 store，组件不会重新渲染，所有页面始终显示空状态。

### 核心修复
**HomeView.vue**: 模板中全部改为 `taskStore.todayData`/`authStore.user` 直接引用，利用 Pinia Proxy 的自动响应式
**ProfileView.vue**: `const user = authStore.user` → 模板中 `authStore.user`
**AchieveView.vue**: 已正确使用 `computed()`，无需修改
**LearnView.vue**: 已正确直接访问 store，无需修改

### 数据库种子: DataInitializer.java 🆕

## Bug 修复 (12:57-12:59) — 页面切换后全部空白

### 根因
`LearnView.vue` 模板有**多个根节点**（3个：`v-if` div, `v-else` div, StarAnimation），构成 Vue fragment。
但 `App.vue` 的 `<Transition mode="out-in">` 要求组件必须是**单一根元素**才能执行动画。
Transition 遇到 fragment 时，out-in 模式卡住，导致 `<router-view>` 不再渲染任何组件。

### 修复 (2 处)
1. **LearnView.vue**: 用 `<div class="learn-root">` 包裹所有子节点，转化为单一根元素
2. **App.vue**: 移除 `mode="out-in"`，保留 `name="fade"` 过渡效果

### 验证
- 多轮切换 Home→Learn→Achieve→Profile→Home → **0 errors, 0 warnings**
- npx vite build → 11 JS 文件 (980KB)

## Bug 修复 (June 14 00:43) — 首页显示"该日期暂无学习任务"

### 根因
`KET_daily_plan.json` 只包含 1 天数据（2026-06-13），今天（2026-06-14）数据库无任务，`TaskService.getByDate()` 直接抛异常。

### 修复
`TaskService.getByDate()` 和 `getTaskByDate()` 添加 fallback：当精确日期无任务时，回退到数据库中最新的可用任务（`<= date ORDER BY task_date DESC LIMIT 1`）。

### 验证
- 首页正常渲染：第1天、入门启动主题、4 个模块、0 errors
